Problema Identificado e Solucionado

❌ Problema Anterior:

✅ Solução Implementada:

Adicionada a rota faltante: $router->get('/admin/profiles/{id}/permissions', [ProfilesController::class, 'getPermissions']);

Como Testar

  1. Acesse o sistema SGQ em produção
  2. Vá para Administrativo > Gerenciar Perfis
  3. Clique em "Editar" em qualquer perfil (mesmo com usuários associados)
  4. O formulário deve abrir normalmente sem erro "Erro de conexão"
  5. Você pode editar nome, descrição, status padrão e permissões
  6. Salvar deve funcionar normalmente
